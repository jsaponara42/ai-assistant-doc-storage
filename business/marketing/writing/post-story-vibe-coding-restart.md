## Draft

At some point in a vibe coding project, the direction changed.

Not dramatically. Just enough that the foundation we'd built no longer fit what we were trying to build on top of it. And we had a choice: spend time patching what existed, or start over with the right principles from the beginning.

We restarted. It was the right call.

This goes against every instinct when you've already built something. The existing code feels like a sunk cost worth saving. You think: we're most of the way there, we just need to fix a few things. But "a few things" in a codebase built on the wrong assumptions tends to mean everything.

What AI changes about this calculus is the cost of restarting. It used to be enormous. Weeks of work, gone. Now the restart is significantly cheaper because the build itself is faster. The logic, the structure, the thing you learned from the first attempt — that carries over. The code doesn't have to.

The expensive part of building isn't the code anymore. It's the thinking. And if you did the thinking right the first time, a restart isn't starting from zero. It's starting from the right place with better information.

Know when to cut the rope. The build you're protecting might be the thing slowing you down.
