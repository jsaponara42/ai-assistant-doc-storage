## Draft

I work across multiple projects at any given time. Client work, internal builds, personal stuff. For a long time I managed this badly.

The problem wasn't the tools. It was the mental model.

I used to organize everything by project. All the tasks for client A together, all the tasks for client B together, and so on. It felt logical. It was a mess in practice. Every morning I'd open my task manager and have to make a judgment call about which project deserved my attention today. That call was influenced by whatever I was anxious about, not what actually needed to get done.

The fix was simple. I stopped organizing by project first and started organizing by due date first.

Now every task lives in a project, but I view them all together sorted by when they're due. I don't have to decide which project gets priority. The date decides. I just work down the list.

The project context is still there when I need it. But the daily question of what to do next is already answered before I sit down.

It sounds almost too simple. But the number of decisions you remove from your morning has a compounding effect. Less time deciding, more time doing. And accountability becomes consistent because due dates don't care which project they belong to.
