# ai-assistant-doc-storage

This repository stores markdown notes generated by AI assistants. It is organized so that both humans and AI tools can navigate and contribute to it easily.

---

## Repository Structure

Notes are organized by **context** at the top level, then by **AI assistant** within each context folder:

```
personal/
  chatgpt/
  claude/
  gemini/
  ...
business/
  chatgpt/
  claude/
  gemini/
  ...
```

| Folder | Purpose |
|--------|---------|
| `personal/` | Notes related to personal topics, hobbies, learning, health, finance, etc. |
| `business/` | Notes related to work, projects, clients, strategy, etc. |

Additional top-level folders may be added as new contexts arise (e.g., `research/`, `creative/`).

---

## Instructions for AI Assistants

When creating or updating a note in this repository, follow these guidelines:

### 1. Choose the Correct Folder

- Use `personal/` for anything related to the user's private life, personal goals, or general knowledge.
- Use `business/` for anything work- or business-related.
- Place the file inside a subfolder named after the AI that generated the note (e.g., `personal/claude/`, `business/chatgpt/`).

### 2. File Naming

- Use lowercase, hyphenated filenames that clearly describe the content.
- Format: `YYYY-MM-DD-descriptive-title.md`
- Example: `2026-03-27-meal-planning-tips.md`

### 3. Required Obsidian-Style Frontmatter

Every markdown file **must** begin with a YAML frontmatter block. This ensures compatibility with [Obsidian](https://obsidian.md/) and similar markdown tools.

```yaml
---
title: "Human-readable title of the note"
date: YYYY-MM-DD
tags: [tag1, tag2, tag3]
ai: name-of-ai-assistant
context: personal | business
summary: "One-sentence summary of the note content."
---
```

| Field | Description |
|-------|-------------|
| `title` | A clear, human-readable title. |
| `date` | ISO 8601 date the note was created (`YYYY-MM-DD`). |
| `tags` | A list of relevant topic tags (lowercase, hyphenated). |
| `ai` | The AI assistant that generated the note (e.g., `claude`, `chatgpt`, `gemini`). |
| `context` | Either `personal` or `business` (should match the top-level folder). |
| `summary` | A single sentence summarizing the note. |

### 4. Linking Notes

When a note references another note in this repository, use a relative Obsidian-style wiki link:

```markdown
See also: [[2026-03-27-meal-planning-tips]]
```

### 5. Updating Existing Notes

- Preserve the original frontmatter fields; only add or amend where necessary.
- Append a `updated` field to the frontmatter when revising a note:

```yaml
updated: YYYY-MM-DD
```
